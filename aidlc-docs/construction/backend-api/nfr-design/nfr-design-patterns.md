# Backend API Service NFR 설계 패턴

## 개요
Backend API Service의 비기능적 요구사항을 충족하기 위한 설계 패턴과 아키텍처 결정사항을 정의합니다.

## 복원력 패턴 (Resilience Patterns)

### 1. 회로 차단기 패턴 (Circuit Breaker Pattern)

#### 구현 전략: 표준 구현
- **실패율 기반**: 5분 창에서 50% 이상 실패 시 회로 개방
- **반개방 상태**: 30초 후 테스트 요청 허용
- **동적 타임아웃**: 서비스별 응답 시간 기반 조정

#### 적용 대상
- **Amazon Bedrock**: LLM 추론 호출
- **AgentCore**: 에이전트 실행 요청
- **OpenSearch**: 검색 및 인덱싱 작업
- **외부 MCP 서버**: MCP 도구 호출

#### 구현 세부사항
```python
# 회로 차단기 설정
CIRCUIT_BREAKER_CONFIG = {
    "bedrock": {
        "failure_threshold": 5,
        "recovery_timeout": 30,
        "expected_exception": BedrockException
    },
    "agentcore": {
        "failure_threshold": 3,
        "recovery_timeout": 60,
        "expected_exception": AgentCoreException
    }
}
```

### 2. 재시도 및 백오프 전략

#### 지수 백오프 구현
- **초기 지연**: 100ms
- **최대 지연**: 30초
- **최대 재시도**: 3회
- **지터**: ±25% 랜덤 변동

#### 서비스별 재시도 정책
- **DynamoDB**: 즉시 재시도 (AWS SDK 기본)
- **OpenSearch**: 1초 지연 후 재시도
- **Bedrock**: 2초 지연 후 재시도 (토큰 제한 고려)
- **S3**: 500ms 지연 후 재시도

### 3. 장애 격리 메커니즘

#### 벌크헤드 패턴 (Bulkhead Pattern)
- **연결 풀 분리**: 서비스별 독립적인 연결 풀
- **스레드 풀 분리**: 비동기 작업별 전용 스레드 풀
- **리소스 격리**: CPU/메모리 제한으로 장애 전파 방지

#### 타임아웃 설정
- **API 요청**: 30초 (사용자 대기 한계)
- **파일 처리**: 10분 (대용량 파일 고려)
- **데이터베이스**: 5초 (빠른 응답 필요)
- **외부 서비스**: 15초 (네트워크 지연 고려)

### 4. 우아한 성능 저하 패턴

#### 기능별 성능 저하 전략
- **검색 기능**: 캐시된 결과 반환, 정확도 감소 허용
- **채팅 기능**: 기본 응답 제공, 고급 기능 비활성화
- **파일 처리**: 큐 대기, 처리 지연 알림
- **실시간 기능**: 폴링으로 대체

## 확장성 패턴 (Scalability Patterns)

### 1. 수평 확장 아키텍처

#### 무상태 설계 (Stateless Design)
- **세션 데이터**: Redis 외부 저장
- **파일 업로드**: S3 직접 업로드
- **처리 상태**: DynamoDB 저장
- **캐시 공유**: Redis 클러스터 활용

#### 자동 확장 정책
- **CPU 사용률**: 70% 초과 시 확장
- **메모리 사용률**: 80% 초과 시 확장
- **응답 시간**: 2초 초과 시 확장
- **큐 길이**: 100개 초과 시 확장

### 2. 로드 밸런싱 전략

#### Application Load Balancer 설정
- **라운드 로빈**: 기본 분산 알고리즘
- **헬스 체크**: `/health` 엔드포인트 5초 간격
- **스티키 세션**: WebSocket 연결용
- **SSL 종료**: ALB에서 HTTPS 처리

#### 트래픽 분산 규칙
- **API 요청**: 가중치 기반 라운드 로빈
- **WebSocket**: 연결 유지를 위한 세션 어피니티
- **파일 업로드**: 직접 S3 업로드로 부하 분산
- **배치 작업**: 별도 워커 인스턴스

### 3. 데이터 파티셔닝 설계

#### DynamoDB 파티셔닝 전략
- **워크스페이스 기반**: `workspace_id`를 파티션 키로 사용
- **시간 기반**: 채팅 메시지는 `timestamp` 포함
- **복합 키**: `workspace_id#entity_type#entity_id` 패턴
- **GSI 활용**: 다양한 쿼리 패턴 지원

#### OpenSearch 인덱스 전략
- **워크스페이스별 인덱스**: `kb-{workspace_id}-{date}`
- **시간 기반 롤오버**: 월별 새 인덱스 생성
- **샤드 분산**: 워크스페이스 크기에 따른 샤드 수 조정
- **복제본**: 가용성을 위한 1개 복제본

### 4. 캐싱 전략 및 계층

#### L1 캐시: 애플리케이션 메모리
- **설정 데이터**: 5분 TTL
- **API 스키마**: 10분 TTL
- **메타데이터**: 1분 TTL
- **크기 제한**: 100MB

#### L2 캐시: Redis 클러스터
- **API 응답**: 5분 TTL
- **검색 결과**: 10분 TTL
- **임시 데이터**: 1시간 TTL
- **파일 메타데이터**: 1시간 TTL

## 성능 패턴 (Performance Patterns)

### 1. 비동기 처리 패턴

#### 표준 비동기 구현
- **SQS 기반**: 파일 처리 작업 큐
- **작업 상태 추적**: DynamoDB에 진행 상태 저장
- **결과 알림**: WebSocket으로 실시간 업데이트
- **오류 처리**: DLQ(Dead Letter Queue) 활용

#### 비동기 작업 유형
- **파일 처리**: 업로드 → 추출 → 청킹 → 임베딩 → 인덱싱
- **검색 인덱싱**: 대량 데이터 인덱스 업데이트
- **보고서 생성**: 사용량 통계 및 분석 보고서
- **백업 작업**: 데이터 백업 및 아카이브

### 2. 연결 풀링 및 재사용

#### 최적화된 연결 관리
- **DynamoDB**: 연결 풀 크기 20, 재사용 활성화
- **OpenSearch**: 연결 풀 크기 10, Keep-Alive 설정
- **Redis**: 연결 풀 크기 15, 타임아웃 30초
- **HTTP 클라이언트**: 연결 재사용, 압축 활성화

#### 연결 모니터링
- **활성 연결 수**: CloudWatch 메트릭
- **연결 오류율**: 알림 임계값 5%
- **연결 지연시간**: 평균 응답 시간 추적
- **풀 사용률**: 80% 초과 시 확장

### 3. 배치 처리 최적화

#### 배치 크기 최적화
- **DynamoDB 배치 쓰기**: 25개 항목
- **OpenSearch 벌크 인덱싱**: 100개 문서
- **S3 멀티파트 업로드**: 5MB 청크
- **SQS 배치 전송**: 10개 메시지

#### 병렬 처리 전략
- **파일 청킹**: 동시 5개 청크 처리
- **임베딩 생성**: 배치 크기 50개
- **인덱스 업데이트**: 동시 3개 인덱스
- **데이터 동기화**: 워크스페이스별 병렬

### 4. 응답 압축 및 최적화

#### 압축 설정
- **Gzip 압축**: 1KB 이상 응답
- **압축 레벨**: 6 (성능과 압축률 균형)
- **MIME 타입**: JSON, HTML, CSS, JS
- **캐시 헤더**: 정적 자원 1시간 캐시

#### 응답 최적화
- **페이지네이션**: 기본 20개, 최대 100개
- **필드 선택**: 필요한 필드만 반환
- **지연 로딩**: 관련 데이터 온디맨드 로드
- **ETags**: 변경되지 않은 데이터 캐시 활용

## 보안 패턴 (Security Patterns)

### 1. 인증 및 권한 부여 패턴 → 제거됨

**인증 기능 제거로 인한 변경사항**:
- JWT 토큰 관리 제거
- RBAC 구현 제거
- 사용자 인증 미들웨어 제거
- 권한 검증 로직 제거

**보안 영향 및 대안**:
- API 접근은 네트워크 레벨에서 제어 (VPC, Security Groups)
- 요청 검증을 통한 기본적인 보안 유지
- 속도 제한으로 남용 방지
- 입력 검증으로 악성 요청 차단

### 2. 데이터 암호화 패턴

#### 전송 중 암호화
- **TLS 1.3**: 모든 HTTP 통신
- **HSTS**: 강제 HTTPS 리다이렉션
- **Certificate Pinning**: 인증서 고정
- **Perfect Forward Secrecy**: 키 교환 보안

#### 저장 시 암호화
- **DynamoDB**: AWS KMS 키 사용
- **S3**: SSE-KMS 암호화
- **OpenSearch**: 저장 시 암호화 활성화
- **Redis**: TLS 전송 암호화

### 3. 입력 검증 및 보안 헤더

#### 표준 보안 구현
- **CSRF 토큰**: 상태 변경 요청 보호
- **XSS 방지**: 입력 데이터 이스케이핑
- **SQL 인젝션**: 매개변수화된 쿼리
- **파일 업로드**: 타입 및 크기 검증

#### 보안 헤더 설정
- **X-Content-Type-Options**: nosniff
- **X-Frame-Options**: DENY
- **X-XSS-Protection**: 1; mode=block
- **Referrer-Policy**: strict-origin-when-cross-origin

### 4. 감사 로깅 패턴

#### 로그 수집 전략
- **접근 로그**: 모든 API 요청/응답
- **보안 이벤트**: 비정상적인 요청 패턴
- **데이터 변경**: 중요 데이터 CRUD 작업
- **시스템 이벤트**: 시작/종료, 설정 변경

#### 로그 형식 및 저장
- **구조화된 JSON**: 검색 및 분석 용이
- **상관관계 ID**: 요청 추적 가능
- **CloudWatch Logs**: 중앙 집중식 저장
- **보관 정책**: 90일 보관, 압축 아카이브

## 모니터링 및 관찰성 패턴

### 1. 구조화된 모니터링

#### 메트릭 수집
- **애플리케이션 메트릭**: 응답 시간, 처리량, 오류율
- **비즈니스 메트릭**: 활성 사용자, 파일 처리 수
- **인프라 메트릭**: CPU, 메모리, 네트워크, 디스크
- **커스텀 메트릭**: 도메인별 KPI

#### 로깅 전략
- **JSON 형식**: 구조화된 로그 데이터
- **로그 레벨**: ERROR, WARN, INFO, DEBUG
- **상관관계 ID**: 요청별 고유 식별자
- **컨텍스트 정보**: 사용자, 워크스페이스, 작업 유형

### 2. 알림 및 대시보드

#### 알림 임계값
- **오류율**: 5% 초과 시 즉시 알림
- **응답 시간**: 평균 2초 초과 시 경고
- **가용성**: 99% 미만 시 긴급 알림
- **리소스 사용률**: 80% 초과 시 확장 알림

#### 대시보드 구성
- **시스템 개요**: 전체 시스템 상태
- **성능 메트릭**: 응답 시간, 처리량 추이
- **오류 분석**: 오류 유형별 통계
- **비즈니스 메트릭**: 사용자 활동, 처리량

## 오류 처리 패턴

### 1. 구조화된 오류 처리

#### 오류 분류 체계
- **시스템 오류**: 500번대, 인프라 문제
- **클라이언트 오류**: 400번대, 요청 문제
- **비즈니스 오류**: 422, 비즈니스 규칙 위반
- **외부 서비스 오류**: 502/503, 의존성 문제

#### 오류 응답 형식
```json
{
  "error": {
    "code": "KNOWLEDGE_BASE_NOT_FOUND",
    "message": "지식베이스를 찾을 수 없습니다",
    "details": {
      "knowledge_base_id": "kb-123",
      "workspace_id": "ws-456"
    },
    "correlation_id": "req-789"
  }
}
```

### 2. 복구 및 보상 메커니즘

#### 자동 복구 전략
- **프로세스 재시작**: 메모리 누수 감지 시
- **연결 재설정**: 네트워크 오류 시
- **캐시 무효화**: 데이터 불일치 감지 시
- **큐 재처리**: 메시지 처리 실패 시

#### 보상 트랜잭션
- **파일 처리 실패**: 부분 처리 데이터 정리
- **인덱스 업데이트 실패**: 이전 상태로 롤백
- **결제 처리 실패**: 예약 리소스 해제
- **사용자 등록 실패**: 생성된 리소스 정리
