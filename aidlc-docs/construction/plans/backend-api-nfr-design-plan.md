# Backend API Service NFR 설계 계획

## NFR 요구사항 분석 요약

### 확장성 요구사항
- **동시 사용자**: 50명
- **일일 API 호출**: 100,000건
- **자동 확장**: FastAPI 인스턴스 2-10개
- **데이터베이스**: DynamoDB 온디맨드 모드

### 성능 요구사항
- **API 응답**: 평균 1초, 95th percentile 3초
- **파일 처리**: 10MB 파일 10분 이내
- **실시간 채팅**: 평균 3초, 최대 10초
- **데이터베이스**: DynamoDB 50ms, OpenSearch 300ms

### 가용성 요구사항
- **가동시간**: 99.5% (월 3.6시간 다운타임)
- **복구 목표**: RTO 30분, RPO 5분
- **다중 AZ**: 최소 2개 가용 영역

### 보안 요구사항
- **암호화**: TLS 1.3, AES-256
- **인증**: JWT 토큰 기반
- **접근 제어**: RBAC
- **감사 로깅**: 모든 API 호출 기록

### 기술 스택 결정
- **프레임워크**: FastAPI
- **데이터베이스**: DynamoDB + OpenSearch
- **캐시**: Redis (ElastiCache)
- **저장소**: S3
- **메시지 큐**: SQS + SNS
