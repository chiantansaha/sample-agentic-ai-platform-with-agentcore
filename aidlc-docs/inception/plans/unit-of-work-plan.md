## 분해 결정 질문

### 질문 1: 배포 모델 선택
복잡한 agentic AI 플랫폼의 배포 모델을 어떻게 구성하시겠습니까?

A) 마이크로서비스 아키텍처 (각 도메인이 독립 서비스)
B) 모듈러 모놀리스 (단일 애플리케이션, 도메인별 모듈)
C) 하이브리드 접근법 (핵심은 모놀리스, 특정 기능은 분리)
D) 서비스별 분리 (프론트엔드, API, 처리 서비스 분리)
E) 기타 (아래 [Answer]: 태그 뒤에 설명해주세요)

[Answer]: B

### 질문 2: 프론트엔드 단위 구성
React 프론트엔드 애플리케이션을 어떻게 단위화하시겠습니까?

A) 단일 SPA (모든 기능이 하나의 애플리케이션)
B) 마이크로 프론트엔드 (도메인별 독립 앱)
C) 기능별 모듈 (단일 앱 내 독립 모듈)
D) 페이지별 분할 (라우트 기반 코드 분할)
E) 기타 (아래 [Answer]: 태그 뒤에 설명해주세요)

[Answer]: A

### 질문 3: 백엔드 서비스 분해
FastAPI 백엔드를 어떻게 단위로 분해하시겠습니까?

A) 도메인별 독립 서비스 (MCP, KB, Agent, Playground 각각 서비스)
B) 기능별 서비스 (관리 API, 실행 API, 파일 처리 API)
C) 계층별 서비스 (API Gateway, 비즈니스 로직, 데이터 액세스)
D) 단일 API 서비스 (모든 도메인이 하나의 FastAPI 앱)
E) 기타 (아래 [Answer]: 태그 뒤에 설명해주세요)

[Answer]: D

### 질문 4: 파일 처리 단위 구성
벡터 처리 및 파일 처리를 어떻게 단위화하시겠습니까?

A) 도메인별 Lambda 함수 (KB 전용, MCP 전용 등)
B) 기능별 Lambda 함수 (파일 업로드, 벡터 생성, 인덱싱)
C) 단일 처리 서비스 (모든 파일 처리를 하나의 서비스)
D) 파이프라인 기반 (Step Functions로 오케스트레이션)
E) 기타 (아래 [Answer]: 태그 뒤에 설명해주세요)

[Answer]: A

### 질문 5: 데이터 소유권 전략
데이터 소유권과 접근을 어떻게 구성하시겠습니까?

A) 도메인별 데이터 소유권 (각 도메인이 자체 데이터 관리)
B) 공유 데이터 레이어 (중앙화된 데이터 서비스)
C) 하이브리드 접근법 (핵심 데이터는 공유, 도메인 데이터는 분리)
D) 이벤트 소싱 (이벤트 기반 데이터 동기화)
E) 기타 (아래 [Answer]: 태그 뒤에 설명해주세요)

### 질문 6: 개발 및 배포 순서
단위들의 개발 및 배포 순서를 어떻게 결정하시겠습니까?

A) 의존성 기반 순서 (의존성이 적은 것부터)
B) 비즈니스 가치 기반 (사용자 가치가 높은 것부터)
C) 기술적 위험 기반 (위험이 높은 것부터)

### 질문 7: 통합 전략
단위 간 통합을 어떻게 관리하시겠습니까?

A) API 기반 통합 (REST/GraphQL API)

C) 직접 통합 (라이브러리/모듈 import)

### 질문 8: 코드 구조 조직 (Greenfield)

C) 하이브리드 (핵심은 모노레포, 일부는 분리)
D) 워크스페이스 기반 (패키지 매니저 워크스페이스)
