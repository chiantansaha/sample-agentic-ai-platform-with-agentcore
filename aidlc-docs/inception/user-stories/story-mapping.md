# 페르소나-스토리 매핑

## 페르소나별 주요 스토리 매핑

### 1. 최종 사용자 (김지현 - AI 연구원) - Primary Persona
**주요 스토리**:
- 4.1 에이전트 선택
- 4.2 에이전트 준비  
- 4.3 채팅 인터페이스
- 4.4 에이전트 테스트
- 4.5 새로운 채팅 세션 시작

**사용 패턴**: 플레이그라운드 중심의 에이전트 테스트 및 평가

### 2. 에이전트 개발자 (박민수 - AI 엔지니어)
**주요 스토리**:
- 3.1 에이전트 조회 및 검색
- 3.2 에이전트 상세 조회
- 3.3 에이전트 생성
- 3.4 에이전트 배포 준비
- 3.5 에이전트 수정
- 3.6 에이전트 재배포
- 3.7 에이전트 비활성화

**보조 스토리** (도구 선택을 위해):
- 1.1 MCP 조회 및 검색 (도구 선택 시)
- 2.1 지식베이스 조회 및 검색 (도구 선택 시)

**사용 패턴**: 에이전트 전체 생명주기 관리

---

### 3. MCP/KB 개발자 (이수진 - 백엔드 개발자)
**주요 스토리**:
- 1.1 MCP 조회 및 검색
- 1.2 MCP 상세 조회

- 1.4 내부 MCP 등록
- 1.5 내부 MCP 생성

---

### 4. 플랫폼 관리자 (최영호 - DevOps 엔지니어)
**주요 스토리** (모든 스토리에 대한 관리 권한):
- 모든 MCP 관리 스토리 (1.1-1.7)
- 모든 지식베이스 관리 스토리 (2.1-2.4)
- 모든 에이전트 관리 스토리 (3.1-3.7)
- 모든 플레이그라운드 스토리 (4.1-4.5)

**사용 패턴**: 전체 플랫폼 운영 및 사용자 지원

## 스토리 우선순위 (페르소나 기반)

### 높은 우선순위 (Primary Persona 중심)
1. **플레이그라운드 스토리** (4.1-4.5) - 최종 사용자 핵심 기능
2. **에이전트 생성 스토리** (3.3, 3.4) - 에이전트 개발자 핵심 기능

### 중간 우선순위 (Supporting Features)
3. **지식베이스 생성** (2.2) - 에이전트에 필요한 지식 제공
4. **MCP 등록** (1.3, 1.4, 1.5) - 에이전트에 필요한 도구 제공
5. **에이전트 관리** (3.1, 3.2, 3.5-3.7) - 에이전트 생명주기 관리

### 낮은 우선순위 (Management Features)
6. **조회 및 검색 기능** (1.1, 1.2, 2.1, 3.1, 3.2) - 관리 편의성
7. **수정 및 업데이트 기능** (1.7, 2.3) - 유지보수 기능
8. **활성화/비활성화 기능** (1.6, 2.4, 3.7) - 상태 관리

---

## 페르소나 간 상호작용

### 협업 플로우
1. **MCP/KB 개발자** → 도구 및 지식 자원 생성
2. **에이전트 개발자** → 생성된 자원을 활용하여 에이전트 구성
3. **최종 사용자** → 완성된 에이전트를 플레이그라운드에서 활용
4. **플랫폼 관리자** → 전체 프로세스 지원 및 문제 해결

### 의존성 관계
- 에이전트 생성 (3.3) ← MCP 등록 (1.3-1.5) + 지식베이스 생성 (2.2)
- 에이전트 테스트 (4.4) ← 에이전트 배포 (3.4)
- 모든 관리 기능 ← 조회 및 검색 기능
