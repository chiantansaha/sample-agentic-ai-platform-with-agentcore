# 사용자 여정 매핑

## 1. MCP 관리 여정 (MCP/KB 개발자 중심)

### 외부 MCP 등록 여정
1. **발견**: 사용하고 싶은 외부 MCP 식별
2. **정보 수집**: MCP 엔드포인트, 인증 방식 확인
3. **등록**: 플랫폼에서 MCP명, 설명, 엔드포인트, 인증 정보 입력
4. **검증**: 등록된 MCP 연결 테스트
5. **활성화**: MCP 활성화하여 사용 가능 상태로 변경
6. **활용**: 에이전트 개발자가 해당 MCP를 도구로 선택 가능

### 내부 MCP 개발 여정
1. **설계**: MCP 기능 및 API 명세 설계
2. **개발**: 컨테이너 이미지 또는 API 구현
3. **등록**: 플랫폼에 컨테이너 이미지 업로드 또는 API 정보 입력
4. **배포**: 런타임에 MCP 배포
5. **테스트**: MCP 기능 동작 확인
6. **버전 관리**: 업데이트 및 버전 관리

---

## 2. 지식베이스 관리 여정 (MCP/KB 개발자 중심)

### 지식베이스 생성 여정
1. **계획**: 지식베이스 주제 및 범위 결정
2. **콘텐츠 준비**: 마크다운 문서 작성 및 정리
3. **생성**: 플랫폼에서 KB명, 설명 입력
4. **업로드**: 마크다운 파일 업로드
5. **처리 대기**: 벡터 임베딩 생성 및 OpenSearch 저장
6. **검증**: 지식베이스 검색 및 품질 확인
7. **활성화**: 에이전트에서 사용 가능하도록 활성화

### 지식베이스 유지보수 여정
1. **모니터링**: 지식베이스 사용 현황 및 피드백 수집
2. **업데이트**: 새로운 문서 추가 또는 기존 문서 수정
3. **재처리**: 변경된 내용의 벡터 임베딩 재생성
4. **품질 관리**: 검색 결과 품질 확인 및 개선

---

## 3. 에이전트 생성 여정 (에이전트 개발자 중심)

### 에이전트 개발 여정
1. **요구사항 분석**: 에이전트가 수행할 업무 및 목표 정의
2. **설계**: 필요한 LLM 모델, 도구, 지식베이스 선택
3. **생성**: 플랫폼에서 에이전트 생성 시작
4. **구성**: 
   - LLM 모델 선택 (Bedrock 모델)
   - 인스트럭션 작성
   - 도구 선택 (지식베이스, MCP)
5. **저장**: 에이전트 명세 저장
6. **배포 준비**: 런타임에 에이전트 배포
7. **테스트**: 플레이그라운드에서 에이전트 동작 확인
8. **최적화**: 성능 및 응답 품질 개선
9. **배포 완료**: 최종 사용자가 사용할 수 있도록 배포

### 에이전트 유지보수 여정
1. **모니터링**: 에이전트 사용 현황 및 성능 모니터링
2. **피드백 수집**: 사용자 피드백 및 개선 요청 수집
3. **업데이트**: 인스트럭션, 도구, 모델 업데이트
4. **재배포**: 업데이트된 에이전트 재배포
5. **검증**: 업데이트 후 성능 및 기능 검증

---

## 4. 플레이그라운드 사용 여정 (최종 사용자 중심)

### 에이전트 테스트 여정
1. **에이전트 선택**: 사용 가능한 에이전트 목록에서 선택
2. **준비 대기**: 에이전트 런타임 준비 완료 대기
3. **대화 시작**: 채팅 인터페이스에서 첫 메시지 입력
4. **상호작용**: 
   - 질문 입력 및 에이전트 응답 확인
   - 다양한 시나리오로 에이전트 테스트
   - 응답 품질 및 정확성 평가
5. **결과 분석**: 에이전트 성능 및 유용성 평가
6. **피드백**: 에이전트 개발자에게 개선 사항 피드백

### 연구 활동 여정
1. **연구 목표 설정**: 테스트하고자 하는 AI 기능 정의
2. **에이전트 탐색**: 연구 목적에 적합한 에이전트 찾기
3. **체계적 테스트**: 
   - 표준화된 질문 세트로 테스트
   - 다양한 도메인 질문으로 범용성 확인
   - 에지 케이스 및 오류 상황 테스트
4. **결과 기록**: 테스트 결과 및 관찰 사항 문서화
5. **비교 분석**: 여러 에이전트 간 성능 비교
6. **연구 활용**: 연구 프로젝트에 적합한 에이전트 선택 및 활용

---

## 여정 간 상호작용

### MCP/KB → 에이전트 → 플레이그라운드 플로우
1. **MCP/KB 개발자**가 도구와 지식 자원 생성
2. **에이전트 개발자**가 이를 활용하여 에이전트 구성
3. **최종 사용자**가 플레이그라운드에서 완성된 에이전트 활용
4. **플랫폼 관리자**가 전체 프로세스 지원 및 문제 해결

### 피드백 루프
- 최종 사용자 피드백 → 에이전트 개선
- 에이전트 개발자 요청 → MCP/KB 개선
- 전체 사용자 피드백 → 플랫폼 기능 개선
