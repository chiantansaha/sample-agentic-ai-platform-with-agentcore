# 요구사항 분석

## 의도 분석

### 사용자 요청
"AI-DLC를 사용해서, high-priority-features의 내용을 참고해서 agentic-ai-platform을 개발해줘"

### 요청 유형
**신규 프로젝트** - 에이전틱 AI 플랫폼을 처음부터 구축

### 초기 범위 추정
**시스템 전체** - 여러 통합 컴포넌트를 포함한 완전한 플랫폼 구축:
- MCP (Model Context Protocol) 관리 시스템
- 지식베이스 관리 시스템  
- 에이전트 관리 시스템
- 테스트 및 상호작용을 위한 플레이그라운드

### 초기 복잡도 추정
**복잡함** - 다음을 포함한 다중 컴포넌트 플랫폼:
- 다중 서비스 통합 (AWS Bedrock, OpenSearch, AgentCore)
- 인증 및 권한 부여 시스템
- 파일 업로드 및 처리 기능
- 실시간 채팅 기능
- 컨테이너 배포 및 관리

## 기능 요구사항

고우선순위 기능 명세를 기반으로 한 agentic-ai-platform 요구사항:

### 1. MCP 관리 시스템
- **MCP 조회 및 검색**: MCP명, 엔드포인트, 설명, 버전, 툴 개수, 내부/외부 상태, 활성화/비활성화 상태 표시
- **MCP 검색 기능**: MCP명, 내부/외부 유형, 활성화/비활성화 상태로 검색
- **MCP 활성화/비활성화**: MCP 가용성 토글
- **MCP 상세 조회**: 개요, 툴 목록, 버전 정보 표시
- **외부 MCP 등록**: 엔드포인트, 인증 방식(None/OAuth/API Key)으로 외부 MCP 등록
- **내부 MCP 등록**: 런타임 배포를 위한 컨테이너 이미지 업로드
- **내부 MCP 생성**: API 명세로부터 게이트웨이 생성과 함께 MCP 생성
- **MCP 업데이트**: AWS AgentCore Gateway 버전 정책에 따른 버전 관리

### 2. 지식베이스 관리 시스템
- **KB 조회 및 검색**: 이름, 설명, 활성화/비활성화 상태로 지식베이스 표시
- **KB 생성**: 이름, 설명, 마크다운 파일 업로드로 지식베이스 생성
- **파일 처리**: .md 파일을 S3 버킷에 업로드하고 벡터 저장소(OpenSearch)에 동기화
- **KB 수정**: 자동 동기화와 함께 데이터 소스 업데이트
- **KB 비활성화**: 지식베이스 비활성화

### 3. 에이전트 관리 시스템
- **에이전트 조회 및 검색**: 상세 메타데이터, 버전 정보, 연결된 툴/KB와 함께 에이전트 표시
- **에이전트 생성 (플레이그라운드)**: 
  - LLM 모델 선택 (Bedrock 모델만)
  - 인스트럭션 작성
  - 툴 선택 (지식베이스, 외부 MCP, 내부 MCP/게이트웨이)
  - 에이전트 명세 저장
  - 런타임에 배포하고 채팅 기능 활성화
- **에이전트 수정**: 버전 관리와 함께 LLM, 인스트럭션, 툴 업데이트
- **에이전트 배포**: 런타임 환경에 준비 및 배포
- **에이전트 비활성화**: 에이전트 비활성화

### 4. 플레이그라운드 시스템
- **에이전트 테스트**: 에이전트 테스트를 위한 대화형 채팅 인터페이스
- **런타임 준비**: 런타임 환경에 에이전트 배포
- **채팅 인터페이스**: 배포된 에이전트와의 실시간 대화

## 비기능 요구사항

### 성능 요구사항
- **응답 시간**: 일반적인 쿼리에 대해 3초 이내 채팅 응답
- **파일 업로드**: 최대 10MB 마크다운 파일 지원
- **동시 사용자**: 플레이그라운드에서 최소 50명의 동시 사용자 지원

### 보안 요구사항
- **인증**: 안전한 사용자 인증 및 세션 관리
- **권한 부여**: MCP/KB/에이전트 관리를 위한 역할 기반 접근 제어
- **API 보안**: 적절한 인증을 통한 안전한 API 엔드포인트
- **데이터 보호**: 전송 중 및 저장 중 민감한 데이터 암호화

### 확장성 요구사항
- **수평 확장**: 웹 애플리케이션 및 API 서비스의 확장 지원
- **스토리지 확장**: 증가하는 지식베이스 콘텐츠 및 벡터 임베딩 처리
- **런타임 확장**: 에이전트 런타임 환경의 동적 확장

### 통합 요구사항
- **AWS Bedrock**: LLM 모델 액세스를 위한 통합
- **AWS AgentCore**: 에이전트 런타임 및 게이트웨이 관리를 위한 통합
- **OpenSearch**: 지식베이스 임베딩을 위한 벡터 저장소
- **S3**: 업로드된 문서를 위한 파일 저장소
- **컨테이너 레지스트리**: 내부 MCP 컨테이너 이미지 저장소

### 사용성 요구사항
- **웹 인터페이스**: 모든 관리 기능을 위한 반응형 웹 애플리케이션
- **직관적 내비게이션**: MCP, KB, 에이전트, 플레이그라운드 섹션 간 명확한 구분
- **실시간 피드백**: 장시간 실행 작업(배포, 파일 처리)에 대한 상태 업데이트

## 기술 아키텍처 고려사항

### 확정된 기술 스택 (사용자 답변 기반)
- **배포 환경**: AWS 클라우드 (관리형 서비스 사용)
- **인증**: 인증 기능 없음 (MVP 접근법)
- **규모**: 부서 수준 (50-200명 사용자, 사내 사용)
- **컨테이너 레지스트리**: AWS ECR
- **모니터링**: 모니터링 기능 없음 (MVP 접근법)
- **벡터 처리**: 실시간 처리 (업로드 시 즉시 임베딩 생성)
- **데이터베이스**: AWS DynamoDB (NoSQL)
- **에이전트 런타임**: AWS AgentCore Runtime (완전 관리형)
- **프론트엔드**: React with TypeScript
- **백엔드**: Python with FastAPI
- **파일 업로드**: 백엔드 API를 통해 S3로 스트리밍
- **채팅 인터페이스**: 기본 요청/응답 채팅 인터페이스

### 시스템 컴포넌트
- **프론트엔드**: React/TypeScript 웹 애플리케이션
- **백엔드 API**: Python FastAPI 기반 RESTful API
- **에이전트 런타임**: AWS AgentCore Runtime
- **게이트웨이 서비스**: AWS AgentCore Gateway
- **파일 처리 서비스**: 문서 업로드 및 벡터 처리 담당
- **데이터베이스**: AWS DynamoDB (메타데이터 저장)
- **벡터 저장소**: OpenSearch (지식베이스 임베딩)
- **파일 저장소**: AWS S3 (문서 저장)
- **컨테이너 레지스트리**: AWS ECR (내부 MCP 이미지)

### 아키텍처 특징
- **서버리스 우선**: AWS 관리형 서비스 최대 활용
- **NoSQL 데이터 모델**: DynamoDB를 활용한 확장 가능한 메타데이터 저장
- **실시간 벡터 처리**: 파일 업로드 시 즉시 임베딩 생성 및 OpenSearch 저장
- **단순한 인증 모델**: MVP를 위한 인증 없는 접근
- **기본 채팅 인터페이스**: 복잡한 실시간 기능 없는 단순한 요청/응답 모델

## 성공 기준
- 사용자가 내부 및 외부 MCP를 성공적으로 등록하고 관리할 수 있음
- 사용자가 자동 벡터 처리와 함께 지식베이스를 생성하고 관리할 수 있음
- 사용자가 선택된 툴과 모델로 에이전트를 생성, 구성, 배포할 수 있음
- 사용자가 플레이그라운드 인터페이스를 통해 배포된 에이전트와 상호작용할 수 있음
- 모든 작업이 허용 가능한 성능 임계값 내에서 완료됨
- 시스템이 높은 가용성과 보안 표준을 유지함
